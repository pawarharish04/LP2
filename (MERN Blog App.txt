Install MongoDB
sudo apt-get update
sudo apt-get install -y gnupg curl

curl -fsSL https://www.mongodb.org/static/pgp/server-8.0.asc | \
sudo gpg -o /usr/share/keyrings/mongodb-server-8.0.gpg --dearmor

echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg ] https://repo.mongodb.org/apt/ubuntu noble/mongodb-org/8.0 multiverse" | \
sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list

sudo apt-get update
sudo apt-get install -y mongodb-org
Fix MongoDB Issue (Kernel Compatibility)

This was the important fix:

sudo mkdir -p /etc/systemd/system/mongod.service.d
sudo nano /etc/systemd/system/mongod.service.d/override.conf

Add:

[Service]
Environment="GLIBC_TUNABLES=glibc.pthread.rseq=0"

Then:

sudo systemctl daemon-reload
sudo systemctl restart mongod
sudo systemctl enable mongod
sudo systemctl status mongod

Important AWS Security Rule

You mentioned you also added inbound rule for port 5000.

In AWS EC2 Security Group:

Add inbound rules:

Type	Port	Source
HTTP	80	Anywhere
Custom TCP	5000	Anywhere
SSH	22	My IP
